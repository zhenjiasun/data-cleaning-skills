# Data Cleaning & Feature Selection

## 🚨 Critical Issue: Target Leakage

The biggest problem here isn't missing values or formatting — it's that **6 of your columns contain information from the future** relative to your prediction point (time of application). Using them would give you inflated metrics that collapse in production.

### Columns to DROP

| Column | Reason | Severity |
|---|---|---|
| `loan_status` | Post-outcome label. Directly encodes the target. | 🔴 Fatal |
| `total_payments_made` | Only exists after disbursement and repayment begins. | 🔴 Fatal |
| `latest_fico` | Refreshed monthly *after* origination. A defaulting borrower's FICO craters, so this basically **is** the answer. | 🔴 Fatal |
| `collections_calls` | Only generated *because* the borrower defaulted. This is a **consequence** of the target, not a predictor. | 🔴 Fatal |
| `decision_date` | Not available at prediction time (application submission). | 🟡 Leak |
| `application_id` | Identifier, no predictive value. | ⚪ Irrelevant |
| `is_default` | This is your **label** — keep it as `y`, not as a feature. | — |

> **Why this matters:** Features like `collections_calls` and `latest_fico` will give you an AUC near 0.99 in testing but your model will be useless in production because those values don't exist when you need the prediction.

## Train / Test Split

Because your data is **temporal** (loan applications over time), a random split would let the model train on future data to predict the past. Use a **time-based split** instead:

```python
df = df.sort_values('application_date')

cutoff = df['application_date'].quantile(0.8)  # ~80/20

train = df[df['application_date'] <= cutoff]
test  = df[df['application_date'] >  cutoff]

X_train = train[feature_cols]
y_train = train['is_default']
X_test  = test[feature_cols]
y_test  = test['is_default']
```

> **Why not `train_test_split(random_state=42)`?** A random split lets the model see 2023-06 applications during training and predict 2023-03 applications during testing. That doesn't match production, where you always predict *forward* in time.
