#### 2. 🚨 DST Transition Day (March 10, 2024)
US clocks sprang forward at 2:00 AM on this exact date (EST → EDT, UTC-5 → UTC-4). This is the single most dangerous day of the year for naive timestamp handling. The fact that your Sunday rows fall exactly here strongly suggests a **timezone conversion error shifted dates or created ghost bars**.

### Recommended Cleaning (Python)

```python
import pandas as pd

df = pd.read_csv("soxl_bars.csv")

# ── Step 1: Fix row 8's timestamp column (epoch leaked into string field)
# Not critical because we'll derive everything from event_time, but log it
print("Format anomalies in 'timestamp':")
print(df[~df["timestamp"].str.contains("-", na=False)])

# ── Step 2: Use event_time epoch as source of truth
df["bar_time_utc"] = pd.to_datetime(df["event_time"], unit="s", utc=True)

# ── Step 3: Convert to Eastern — USE tz_convert, NOT tz_localize
#    tz_localize would STAMP these UTC times as ET without converting → hours of silent error
df["bar_time_et"] = df["bar_time_utc"].dt.tz_convert("US/Eastern")

# ── Step 4: Flag and remove weekend bars
df["day_of_week"] = df["bar_time_et"].dt.day_name()
weekend_mask = df["day_of_week"].isin(["Saturday", "Sunday"])
print(f"\n⚠️  Dropping {weekend_mask.sum()} weekend rows:")
print(df.loc[weekend_mask, ["bar_time_utc", "bar_time_et", "day_of_week"]])
df = df[~weekend_mask].copy()

# ── Step 5: Flag bars outside regular session (09:30–16:00 ET)
market_open = df["bar_time_et"].dt.time >= pd.Timestamp("09:30").time()
market_close = df["bar_time_et"].dt.time <= pd.Timestamp("16:00").time()
outside_rth = ~(market_open & market_close)
if outside_rth.any():
    print(f"\n⚠️  {outside_rth.sum()} bars outside regular trading hours:")
    print(df.loc[outside_rth, ["bar_time_et", "close"]])

# ── Step 6: Validate epoch-vs-string consistency on remaining rows
df["timestamp_parsed"] = pd.to_datetime(df["timestamp"], errors="coerce", utc=True)
df["epoch_vs_string_delta"] = (df["bar_time_utc"] - df["timestamp_parsed"]).dt.total_seconds()
mismatches = df[df["epoch_vs_string_delta"].abs() > 0]
if not mismatches.empty:
    print(f"\n⚠️  {len(mismatches)} rows where epoch ≠ string timestamp:")
    print(mismatches[["timestamp", "bar_time_utc", "epoch_vs_string_delta"]])

# ── Step 7: Drop redundant/ambiguous columns, keep clean ones
df = df[["bar_time_utc", "bar_time_et", "ticker", "close", "volume"]].copy()
df["bar_date_et"] = df["bar_time_et"].dt.date
df["bar_hour_et"] = df["bar_time_et"].dt.hour

print("\n✅ Cleaned dataframe:")
print(df.to_string(index=False))
```

### TL;DR — What to trust

| Source | Verdict |
|---|---|
| `event_time` (epoch) | **Best available**, but March 11 epochs may carry a 1-hour DST bug — cross-check against your yfinance pull |
| `timestamp` (string) | **Do not trust.** Mixed format, missing tz, contradicts epoch on multiple rows |

**Cardinal rule for your yfinance bars:** yfinance returns timestamps in UTC. Always use `tz_convert("US/Eastern")`, never `tz_localize`. Localizing silently re-labels UTC values as Eastern, shifting every bar by 4–5 hours depending on DST.
